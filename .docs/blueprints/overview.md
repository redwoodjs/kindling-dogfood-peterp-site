## 2000ft View

This repository is a kindling dogfood fork of [peterp/peterp.github.io](https://github.com/peterp/peterp.github.io) — Peter Pistorius's personal site. It exists as a live test target for the [kindling](https://github.com/redwoodjs/kindling) agentic harness: issues, pull requests, and commits here are produced by kindling agents as part of dogfooding the system. The current site is a single static HTML file (`index.html`) served via GitHub Pages. An active migration epic (issue #8) tracks its conversion to a RedwoodSDK (rwsdk) app running on Cloudflare Workers.

---

## System Flow

The current site has no build step. The entry point is `index.html` at the repository root — a self-contained HTML file with inline styles and hardcoded content. GitHub Pages serves it directly. There is no JavaScript framework, no bundler, and no server-side logic in the current state.

The planned post-migration flow (via the rwsdk epic) will be: source code in a scaffolded RedwoodSDK project (expected under a `web/` directory or similar) → Vite build → Cloudflare Workers deployment triggered by a GitHub Actions workflow on push to `master`.

A stub CI workflow already exists at `.github/workflows/ci.yml` that runs on PRs and pushes to master but currently does nothing (single `true` step).

## Key Abstractions

**index.html — The Entire Current Site**  
A single 40-line HTML file containing all markup, inline CSS, and content for peterp.org. It has no dependencies, no build step, and no JavaScript. This file is what the rwsdk migration epic aims to replace with a structured React application.

**Kindling Board (`.kindling/board/`)**  
The kindling harness manages tasks as files in a state-machine directory structure. Subdirectories represent task states: `inbox`, `todo`, `doing`, `in-review`, `done`, `archived`, `dormant`. Agents move task files between these directories as work progresses. This is the primary task-tracking mechanism for automated agent work on this repo.

**rwsdk Migration Epic (issue #8)**  
The governing work item for the site's technical evolution. It defines seven sequential sub-issues (#1–#7) covering: scaffolding a new RedwoodSDK project, porting HTML content into React components, migrating inline styles, verifying local dev, configuring Wrangler for Cloudflare Workers, adding a CI deploy workflow, and installing agent-ci for local workflow validation. All sub-issues are currently open.

**`.docs/` — Agent Research Artifacts**  
A directory for agent-produced documentation. Not a standard project convention from the upstream site — introduced by the kindling harness. Currently contains the rwsdk epic investigation report and this blueprint.
