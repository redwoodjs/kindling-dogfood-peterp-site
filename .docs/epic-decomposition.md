# Epic Decomposition: Migrate peterp.org to RedwoodSDK

**Epic Issue:** [#8](https://github.com/redwoodjs/kindling-dogfood-peterp-site/issues/8)
**Progress Board:** [#59](https://github.com/redwoodjs/kindling-dogfood-peterp-site/issues/59)
**Upstream:** peterp/peterp.github.io#1

## Overview

This epic migrates the peterp.org personal site from a static single-file `index.html` hosted on GitHub Pages to a RedwoodSDK (rwsdk) application running on Cloudflare Workers. RedwoodSDK is a Vite-based React framework that enables RSC, server functions, and realtime capabilities.

## Sub-Issue Decomposition

### #1 — Scaffold new RedwoodSDK project
**State:** OPEN
**Protocol:** `scaffold`

**Description:** Bootstrap a new RedwoodSDK project using `pnpm create rwsdk`. The scaffolded app must build locally, with `pnpm install` and `pnpm dev` both succeeding. A `pnpm-lock.yaml` must be committed (no npm/yarn lockfiles).

**Task brief:** Scaffold a new RedwoodSDK project via `pnpm create rwsdk`, verify `pnpm dev` serves on localhost:5173, and commit the result with `pnpm-lock.yaml`.

---

### #2 — Port index.html content into RedwoodSDK routes/components
**State:** OPEN
**Protocol:** `port`

**Description:** Convert the semantic HTML content from the existing `index.html` (bio paragraph, social links, side projects list) into RedwoodSDK route files and React components. The home page must render the same content as the current site, preserving all external links.

**Task brief:** Port the bio, social links, and side projects from `index.html` into rwsdk React components, maintaining content parity.

---

### #3 — Port styles to RedwoodSDK app
**State:** OPEN
**Protocol:** `port`

**Description:** Extract the inline `<style>` block from `index.html` (max-width, line-height, padding) and integrate it into the rwsdk app using whichever styling approach rwsdk defaults to (CSS module or global stylesheet). Visual output must match the current site at equivalent viewport sizes.

**Task brief:** Move inline CSS from `index.html` into the rwsdk app's styling system, ensuring visual parity with the original.

---

### #4 — Verify local dev for rwsdk app
**State:** OPEN
**Protocol:** `fix`

**Description:** A verification checkpoint: confirm `pnpm dev` boots cleanly, all routes render correctly, there are no console errors, and hot reload works as expected. This is a sanity check after the content and style porting is complete.

**Task brief:** Run `pnpm dev` and verify all routes render without console errors and hot reload works.

---

### #5 — Add wrangler.toml / Worker config
**State:** OPEN
**Protocol:** `infra`

**Description:** Ensure the Cloudflare Worker configuration file (`wrangler.toml` or `wrangler.jsonc`) generated by rwsdk is committed with the correct worker name, compatibility date, and any required bindings. Verify `wrangler deploy --dry-run` succeeds locally.

**Task brief:** Commit wrangler.toml with correct worker config and verify `wrangler deploy --dry-run` passes.

---

### #6 — Add CI deploy workflow (GitHub Actions → Cloudflare Workers)
**State:** OPEN
**Protocol:** `infra`

**Description:** Create a GitHub Actions workflow that triggers on push to `master` and deploys the rwsdk app to Cloudflare Workers via `pnpm release` (or `wrangler deploy`). The workflow should use `pnpm/action-setup`, enable pnpm caching, and reference a `CLOUDFLARE_API_TOKEN` repository secret.

**Task brief:** Create `.github/workflows/deploy.yml` that deploys to Cloudflare Workers on push to master using pnpm and CLOUDFLARE_API_TOKEN secret.

---

### #7 — Install agent-ci for local GitHub Actions validation
**State:** OPEN
**Protocol:** `infra`

**Description:** Install RedwoodJS's agent-ci tool (`@redwoodjs/agent-ci`) as a dev dependency to enable local validation of GitHub Actions workflows before pushing. Confirm the deploy workflow can be run locally end-to-end. Install the agent skill and add documentation instructing agents to validate workflow changes via agent-ci.

**Task brief:** Add `@redwoodjs/agent-ci` to devDependencies, verify local workflow execution works, and document agent-ci usage in CLAUDE.md or AGENTS.md.

---

## Execution Dependency Graph

```
#1 (scaffold)
    ↓
#2 (port content) ─┬─ #3 (port styles)
                   ↓
                  #4 (verify local dev)
                   ↓
#5 (wrangler config) ─┬─ #7 (agent-ci)
                      ↓
                     #6 (CI deploy workflow)
```

**Phase 1:** #1 — Foundation must exist before any porting work
**Phase 2:** #2, #3 — Content and style porting can proceed in parallel
**Phase 3:** #4 — Validation depends on Phase 2 completion
**Phase 4:** #5, #7 — Infrastructure work can proceed in parallel
**Phase 5:** #6 — CI workflow depends on wrangler config and ideally agent-ci for validation

## Tooling Constraint

All sub-issues must use `pnpm` for package management and scripts (not npm/yarn).
